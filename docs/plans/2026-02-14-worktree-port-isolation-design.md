# Worktree Port Isolation & Project-Centric UI Design

Date: 2026-02-14
Status: Draft

## Problem

Multiple git worktrees running simultaneously compete for the same ports (frontend, backend, Redis DB).
Currently, port isolation is handled manually or via project-specific scripts (e.g., MediaHub's `worktree-manager.sh`).
The Web UI is organized around tmux sessions, making it hard to manage project-level configuration like environment variables and service ports.

## Goals

1. Automatic port isolation when creating worktrees via Web UI
2. Project-level environment variables (like GitHub Secrets & Variables), editable in Web UI
3. Auto-generate `.worktree.env` per worktree, auto-sync on variable changes
4. AI agent activation: MEMORY.md pointer so Claude reads `.worktree.env` automatically
5. Project-centric UI navigation (project -> worktrees/variables/sessions)

## Non-Goals

- Template rendering for framework-specific files (e.g., `frontend/.env.local`) -- projects handle this themselves
- Multi-user / team collaboration
- Remote worktree management

## Architecture

### Data Flow

```
Web UI (project settings)
    │
    ▼
ai-tracker server (Rust/axum)
    │
    ├── Central SQLite (new tables)
    │   ├── project_env_vars      -- custom key-value pairs
    │   ├── project_services      -- service definitions + base ports
    │   └── worktree_slots        -- slot allocation registry
    │
    ├── Generate .worktree.env    -- written to worktree root
    │
    └── Write MEMORY.md pointer   -- one-time, per worktree
```

### Relationship to Existing Systems

```
Unchanged:
  .aitracker/tracker.db       -- per-project local data (history, notes, commits)
  ~/.config/agent-tracker/     -- central config, web dist, scripts
  tmux session management      -- create/destroy/resume windows unchanged

New (additive):
  Central SQLite new tables    -- project_env_vars, project_services, worktree_slots
  .worktree.env generation     -- written to worktree directories
  Project-centric UI layer     -- groups tmux sessions under projects
```

## Data Model

### project_env_vars

Custom key-value pairs per project. Fully user-defined, no predefined keys.

```sql
CREATE TABLE IF NOT EXISTS project_env_vars (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_name TEXT NOT NULL,          -- project identifier (tmux session name)
    key         TEXT NOT NULL,
    value       TEXT NOT NULL DEFAULT '',
    is_secret   INTEGER NOT NULL DEFAULT 0,  -- 1 = masked in UI
    sort_order  INTEGER NOT NULL DEFAULT 0,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now')),
    UNIQUE(session_name, key)
);
```

Examples for mediahub:
| key | value | is_secret |
|-----|-------|-----------|
| SUPABASE_URL | http://127.0.0.1:54321 | 0 |
| SUPABASE_ANON_KEY | sb_publishable_xxx | 0 |
| SUPABASE_SERVICE_ROLE_KEY | sb_secret_xxx | 1 |
| REDIS_HOST | 127.0.0.1 | 0 |
| REDIS_PORT | 16379 | 0 |
| DOWNLOAD_PATH | /Volumes/sources/MediaHub.library | 0 |

### project_services

Service definitions with base port/value for slot-based allocation.

```sql
CREATE TABLE IF NOT EXISTS project_services (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    session_name TEXT NOT NULL,
    service_name TEXT NOT NULL,           -- user-defined: "frontend", "backend", etc.
    base_value   INTEGER NOT NULL,        -- base port or db index
    value_type   TEXT NOT NULL DEFAULT 'port',  -- 'port' or 'db_index'
    env_key      TEXT NOT NULL,           -- variable name in .worktree.env
    sort_order   INTEGER NOT NULL DEFAULT 0,
    UNIQUE(session_name, service_name)
);
```

Examples for mediahub:
| service_name | base_value | value_type | env_key |
|--------------|-----------|------------|---------|
| frontend | 5175 | port | FRONTEND_PORT |
| backend | 8080 | port | BACKEND_PORT |
| redis_db | 0 | db_index | REDIS_DB |

### worktree_slots

Tracks which slot is assigned to which worktree.

```sql
CREATE TABLE IF NOT EXISTS worktree_slots (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    session_name  TEXT NOT NULL,
    slot          INTEGER NOT NULL,       -- 0 = main, 1~15 = worktrees
    branch        TEXT NOT NULL,
    worktree_path TEXT,                   -- absolute path on disk
    created_at    TEXT NOT NULL DEFAULT (datetime('now')),
    UNIQUE(session_name, slot),
    UNIQUE(session_name, branch)
);
```

## .worktree.env Generation

### Format

```env
# ============================================
# Auto-generated by ai-tracker (do not edit)
# ============================================
# Project: mediahub
# Branch:  feature/payment-system
# Slot:    2
# Updated: 2026-02-14 15:30:00

# Project Variables
SUPABASE_URL=http://127.0.0.1:54321
SUPABASE_ANON_KEY=sb_publishable_xxx
SUPABASE_SERVICE_ROLE_KEY=sb_secret_xxx
REDIS_HOST=127.0.0.1
REDIS_PORT=16379
DOWNLOAD_PATH=/Volumes/sources/MediaHub.library

# Worktree Isolated (slot 2)
WORKTREE_NAME=feature-payment-system
WORKTREE_SLOT=2
FRONTEND_PORT=5177
BACKEND_PORT=8082
REDIS_DB=2
```

### Generation Logic

```
fn generate_worktree_env(session_name, slot, branch, worktree_path):
    vars = db.get_project_env_vars(session_name)     -- all key-value pairs
    services = db.get_project_services(session_name)  -- service definitions

    output = header_comment(session_name, branch, slot)

    # Section 1: Project variables (sorted by sort_order)
    for var in vars:
        output += f"{var.key}={var.value}"

    # Section 2: Worktree isolated values
    output += f"WORKTREE_NAME={sanitize(branch)}"
    output += f"WORKTREE_SLOT={slot}"
    for svc in services:
        calculated = svc.base_value + slot
        output += f"{svc.env_key}={calculated}"

    write_file(worktree_path / ".worktree.env", output)
```

### Slot Allocation

```
fn next_available_slot(session_name) -> u16:
    used_slots = db.get_used_slots(session_name)  -- Set<u16>
    for slot in 1..=15:
        if slot not in used_slots:
            return slot
    error("All 15 slots used")
```

Slot 0 is reserved for the main branch (optional, auto-registered on first use).

### Auto-Sync on Variable Change

When any `project_env_vars` or `project_services` row is updated:

```
fn sync_all_worktree_envs(session_name):
    slots = db.get_worktree_slots(session_name)
    for slot in slots:
        if path_exists(slot.worktree_path):
            generate_worktree_env(session_name, slot.slot, slot.branch, slot.worktree_path)
```

Triggered by: POST/PUT/DELETE on env vars or services API endpoints.

## AI Agent Activation

### MEMORY.md (one-time write)

When creating a worktree, ai-tracker writes to the corresponding Claude Code MEMORY.md:

Path pattern: `~/.claude/projects/-{path-encoded-worktree-dir}/memory/MEMORY.md`

Content appended (if not already present):

```
## Dev Environment
When starting dev servers or configuring ports, read .worktree.env in the project root for isolated port assignments.
```

This is a pointer, not data. The AI reads `.worktree.env` at runtime for actual values.

### Why pointer, not data

- `.worktree.env` is auto-synced when variables change
- MEMORY.md doesn't need updating when ports/variables change
- Single source of truth: `.worktree.env`

## API Endpoints

### Project Environment Variables

```
GET    /api/project/env-vars?session_name=mediahub
POST   /api/project/env-vars                          -- { session_name, key, value, is_secret? }
PUT    /api/project/env-vars/:id                      -- { key?, value?, is_secret?, sort_order? }
DELETE /api/project/env-vars/:id
```

All mutation endpoints trigger `sync_all_worktree_envs(session_name)`.

### Project Services

```
GET    /api/project/services?session_name=mediahub
POST   /api/project/services                          -- { session_name, service_name, base_value, value_type, env_key }
PUT    /api/project/services/:id                       -- { service_name?, base_value?, value_type?, env_key? }
DELETE /api/project/services/:id
```

All mutation endpoints trigger `sync_all_worktree_envs(session_name)`.

### Worktree Slots

```
GET    /api/project/worktree-slots?session_name=mediahub
POST   /api/project/worktree-slots                    -- { session_name, branch }  (auto-allocates slot)
DELETE /api/project/worktree-slots/:id                 -- frees the slot
```

POST returns the allocated slot and generated port values.

### Existing Endpoints (unchanged)

```
POST   /api/workspace/start     -- add: auto-register worktree_slot if not exists
POST   /api/workspace/destroy   -- add: auto-free worktree_slot
POST   /api/workspace/resume    -- unchanged
GET    /api/git/branches         -- unchanged
```

10. **Testing & polish** -- edge cases, error handling

## Edge Cases

- **Worktree deleted outside ai-tracker**: slot remains allocated. Provide "Sync" button to detect and clean stale slots.
- **Port conflict with other projects**: slots are per-project. Cross-project conflict is possible if two projects use same base ports. Acceptable for single-developer use; can add global port registry later if needed.
- **No services defined**: skip .worktree.env generation entirely, behave like current system.
- **Slot exhaustion (>15 worktrees)**: show error in UI, suggest destroying unused worktrees.
- **Secret values in .worktree.env**: written as plain text (local dev files). UI masks them but the file doesn't encrypt. Add `.worktree.env` to project `.gitignore`.
